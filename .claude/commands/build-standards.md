# Выполнение плана: $ARGUMENTS

## Подготовка

1. Прочитай план из `_bmad-output/plans/[область]-standards-plan.md`
2. Убедись что ветка правильная (`git branch --show-current`)
3. Убедись что рабочая директория чистая (`git status`)

## ВАЖНО: не останавливайся

Выполняй ВСЕ шаги плана в одной сессии. Не предлагай "продолжить потом" и не оценивай сложность оставшихся шагов как причину для паузы. Если контекст становится большим — работай дальше, система сама сожмёт историю. Единственная причина для остановки — явная просьба пользователя.

## Выполнение

Для каждого шага из плана:

### Паттерн выполнения шага

1. **Прочитай** текущее состояние файлов из шага
2. **Прочитай** стандарт, на который ссылается шаг
3. **Реализуй** изменения, строго следуя стандарту
4. **Проверь** — запусти проверку, указанную в шаге (билд, тест, линтер)
5. **Закоммить** с описательным сообщением

### Формат коммита
```
refactor([scope]): [описание]

- [деталь 1]
- [деталь 2]

Standards: [ссылка на docs/standards/файл.md]
```

## Если шаг сломал что-то

1. Попробуй исправить ошибку самостоятельно (максимум 3 попытки)
2. Если не получилось — откати шаг (`git checkout -- .`), запиши проблему в `_bmad-output/plans/[область]-standards-blockers.md` и переходи к следующему шагу

## После всех шагов

1. Запусти полный набор проверок (билд + тесты + линтеры)
2. Сохрани итог в `_bmad-output/plans/[область]-standards-result.md`:
   - Выполнено шагов: N из M
   - Коммитов создано: N
   - Пропущенные шаги (с причинами) — если есть
   - Результат финальных проверок: pass/fail
3. Выведи краткое саммари результатов
