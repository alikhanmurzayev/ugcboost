# План исправлений: $ARGUMENTS

## Шаг 1: Загрузи контекст

1. Прочитай отчёт из `_bmad-output/plans/[область]-scout-report.md`. Если файла нет — сначала выполни шаги из `/scout-standards` самостоятельно.
2. Прочитай релевантные стандарты из `docs/standards/`

## Шаг 2: Сгруппируй работу

Раздели все нарушения на логические шаги. Каждый шаг — это одна атомарная единица работы, которую можно выполнить и проверить отдельно.

Принципы группировки:
- Один шаг = один коммит
- Шаги не ломают билд и тесты между собой
- Сначала структурные изменения (перемещение файлов, переименование), потом содержательные
- Зависимые изменения идут последовательно

## Шаг 3: Напиши план

Сохрани план в `_bmad-output/plans/[область]-standards-plan.md`:

```markdown
# План: приведение [область] к стандартам

## Обзор
[1-2 предложения — что будет сделано]

## Шаги

### Шаг 1: [название]
- **Файлы**: список файлов
- **Что делаем**: конкретные изменения
- **Стандарт**: ссылка на docs/standards/
- **Проверка**: как убедиться что шаг выполнен (тест, линтер, билд)

### Шаг 2: [название]
...

## Порядок выполнения
[Если есть зависимости между шагами — описать]

## Проверка после всех шагов
- [ ] Билд проходит
- [ ] Тесты проходят
- [ ] Линтеры проходят
```

## Ша�� 4: Выведи план

Выведи краткое саммари плана (количество шагов, основные области изменений). Полный план — в файле.
