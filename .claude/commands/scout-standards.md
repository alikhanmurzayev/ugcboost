# Разведка по стандартам: $ARGUMENTS

## Шаг 1: Загрузи стандарты

Прочитай стандарты кода из `docs/standards/`, релевантные для указанной области:
- **backend** → все файлы `backend-*.md` + `naming.md` + `security.md`
- **frontend** или **web** → все файлы `frontend-*.md` + `naming.md` + `security.md`
- **tma** → все файлы `frontend-*.md` + `naming.md` + `security.md`
- **infra** или **cicd** → `security.md` + читай `infra/`, `.github/workflows/`
- **all** → все стандарты

Если область не указана — используй **all**.

## Шаг 2: Просканируй код

1. Найди все файлы в указанной области через `git ls-files`
2. Прочитай ключевые файлы (handlers, services, repositories, components, pages)
3. Для каждого файла сверяй с прочитанными стандартами

## Шаг 3: Отчёт

Выведи отчёт в формате:

### Область: [название]

### Нарушения стандартов

Для каждого нарушения:
- **Файл**: `path/to/file:line`
- **Стандарт**: какой именно стандарт нарушен (ссылка на файл в docs/standards/)
- **Что не так**: конкретное описание
- **Как исправить**: конкретное действие

Группируй по приоритету:
1. **Критичные** — ломают архитектуру, безопасность, тестируемость
2. **Важные** — нарушают конвенции, усложняют поддержку
3. **Мелкие** — нейминг, форматирование, стиль

### Статистика
- Всего файлов проверено: N
- Нарушений найдено: N (критичных / важных / мелких)

### Рекомендация
Кратко — с чего начать исправления и в каком порядке.

## Шаг 4: Сохрани отчёт

Сохрани отчёт в `_bmad-output/plans/[область]-scout-report.md` для использования в `/plan-standards`.
