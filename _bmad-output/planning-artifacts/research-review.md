## Market Research: критика

### Критические проблемы

#### 1. "Нулевая конкуренция" — ложный вывод

Отсутствие идентичного продукта ≠ отсутствие конкуренции. Реальные конкуренты UGCBoost:

- Ручной поиск через Instagram DM (бесплатно, контроль, привычно)
- Маркетинговые агентства (Brave Talents Co. и десятки других)
- Внутренние маркетологи брендов с собственными пулами креаторов
- Бартерные сделки в Telegram-чатах

Документ сам упоминает: "Ручной подбор через Instagram DM и Google-формы остаётся доминирующим способом работы". Но вместо анализа **почему** это доминирует и **что нужно для перехода** на платформу, документ переключается на перечисление глобальных платформ, которые вообще не работают в Казахстане.

#### 2. Unit-экономика на непроверенных допущениях

Break-even при 19 активных брендах (38 кампаний/мес, GMV $800) — оптимистично:

- **GMV $800** нереалистичен для КЗ. По данным того же документа, МСБ тратят $450-7,000/год на UGC. Средний GMV одной кампании скорее $200-400.
- **2 кампании/мес/бренд** — нет обоснования. Для казахстанского МСБ это скорее квартальная частота.
- **OPEX $3,000/мес** — не включает зарплаты основателей, юридические расходы, непредвиденное. Реалистичный OPEX: $5,000-8,000/мес.

При пересчёте (GMV $400, 1 кампания/мес, OPEX $5K) break-even: ~125 активных брендов. Совсем другая картина.

#### 3. "Organic inbound = PMF" — опасная подмена

Три разговора с потенциальными клиентами (One Corporate, Beauty by Sofia, First Media Group) — это hypothesis validation, не product-market fit. Не известно:
- Готовы ли они **платить** и сколько?
- Что конкретно хотят — платформу или помощь с блогерами?
- Вернутся ли после первой кампании?

### Серьёзные пробелы

- **Willingness to pay** — нет данных о готовности казахстанских брендов платить за UGC через платформу
- **TAM → SAM → SOM** — отсутствует, инвестор спросит первым
- **Дизинтермедиация** — бренд находит креаторов через платформу и уходит работать напрямую. Этот риск не упомянут
- **LTV:CAC до 108:1** — абсурдная цифра для pre-revenue стартапа, подрывает доверие
- **"Почему сейчас?"** — нет ответа, почему UGC-маркетплейс не появился в КЗ раньше. Возможные причины (маленький рынок, бартер решает всё) не исследованы

### Сомнительные данные

- **"100K-300K микрокреаторов в КЗ"** — экстраполяция глобальных процентов (1-2%) на КЗ. Perfluence за всё время привлекла ~20K авторов — реальность ближе к этой цифре
- **Креаторская экономика "Ближнего Востока и Центральной Азии" $10.93 млрд** — включает ОАЭ, Саудовскую Аравию. Доля КЗ — менее 1-2%
- **"93% рост UGC-креаторов YoY"** — глобальная цифра Collabstr, нерелевантна для КЗ

## Technical Research: критика

### Критические проблемы

#### 1. LiveDune API — ядро продукта на непроверенной гипотезе

"Подтверждённые API-методы" на самом деле из документации Albato (третьи руки), не из LiveDune напрямую. Критический вопрос — можно ли проверять аккаунты, не подключённые к dashboard — скорее всего имеет **отрицательный** ответ. Аналитические платформы обычно требуют добавления аккаунта в систему.

Fallback на Phyllo ($10,000/год) убивает unit-экономику MVP.

#### 2. TrustContract API — не прочитана документация

Документация на Apiary требует JS-рендеринга — фактически не читали. Не известны: эндпоинты, аутентификация, вебхуки, виджеты. Вся секция — набор предположений.

#### 3. Три фронтенда для одного разработчика

TMA (React+Vite) + Dashboard (Next.js) + Landing — нереалистично для команды из одного разработчика. Admin и Brand Dashboard должны быть одним приложением с RBAC.

#### 4. Баг: telegram_id как Integer

`telegram_id: Mapped[int | None] = mapped_column(Integer, ...)` — PostgreSQL Integer это 32-bit (макс. 2,147,483,647). Telegram ID давно превышает этот лимит. Нужен `BigInteger`.

### Ошибки в коде

| Проблема | Где | Правильное решение |
|---|---|---|
| Fernet (AES-128-CBC) вместо обещанного AES-256-GCM | Секция 11.8 | Использовать `cryptography.hazmat` с AES-256-GCM |
| `datetime.utcnow()` deprecated в Python 3.12+ | Секции 9.2, 13 | `datetime.now(UTC)` |
| HS256 для JWT в мультисервисной системе | Секция 9.2 | RS256/ES256 с асимметричными ключами |
| APScheduler без persistent storage | Секция 8.3 | `SQLAlchemyJobStore` или Celery beat |
| CORS для TMA WebView не исследован | Секция 11.9 | Origin в WebView непредсказуем — нужно тестировать |
| Webhook бота на отдельном порту (aiohttp) | Секция 13.2 | Интегрировать в FastAPI через `SimpleRequestHandler` |

### Нереалистичные оценки

- **Pre-dev чеклист 3 недели** → реально 6-8 недель (TrustMe ответит через 1-4 недели, Facebook Verification — 4-6 недель)
- **Инфраструктура ~5,500 KZT/мес** → реально ~40-60K KZT/мес с LiveDune ($50), TrustMe (SMS-подписи), email-сервис, object storage
- **Hetzner Финляндия "80-120ms"** — это один запрос. Загрузка страницы TMA (5-10 API-вызовов) = 400-1200ms на сетевую задержку
- **"100% delivery" уведомлений через Telegram** — неверно, если пользователь заблокировал бота

### Серьёзные пробелы

- **Файловое хранилище** — не проанализировано. UGC = видео. 500 креаторов × 2-3 видео = 50-150 GB. Где хранить? CDN? Транскодирование?
- **Email-отправка** — Brand Dashboard предполагает email, но провайдер, доставляемость, SPF/DKIM не описаны
- **Логистика бартера** — как бренд доставляет товар креатору? Как подтверждается получение? Где это в системе?

## Domain Research: критика

### Критические юридические риски

#### 1. Моральные (неимущественные) права автора — полностью проигнорированы

По ст. 15-16 Закона об авторском праве РК (и Бернской конвенции):

- Право на имя — **неотчуждаемо**
- Право на неприкосновенность произведения — бренд **не может** произвольно монтировать видео
- Право на обнародование — решает автор

"Full Buyout" (раздел 6.1) **юридически невозможен** в полном объёме по казахстанскому праву. Нужно отдельное **согласие** автора на переработку, монтаж, использование фрагментов.

#### 2. НДС-посредничество — риск переквалификации

Схема "UGCBoost как ТОО-посредник с НДС" несёт риски:

- **Притворная сделка** (ст. 160 ГК РК) — если UGCBoost только передаёт деньги без собственных услуг добавленной стоимости
- **Трансфертное ценообразование** (глава 18 НК РК) — при нерыночной комиссии
- **Агент vs комиссионер** — не уточнено, а это принципиально для базы НДС (только комиссия или весь оборот)

#### 3. "Undercover creators" — непроверенная гипотеза

Ст. 6 Закона о рекламе запрещает введение в заблуждение о характере информации. Суть "undercover" — создать впечатление органического контента, когда он таковым не является. Хештег #реклама среди 30 других хештегов может не считаться "clear and conspicuous".

Необходимо заключение казахстанского юриста-практика. "Реализуемо" — это **мнение**, не юридический факт.

#### 4. Бартер = доход в натуральной форме

Получение продукта в бартер подлежит налогообложению. UGCBoost как налоговый агент обязан оценить стоимость и удержать налоги. Невключение — налоговое правонарушение. В документе не раскрыто.

### Серьёзные пробелы

| Пропущенная тема | Почему важно |
|---|---|
| Закон о защите прав потребителей | UGC = рекомендация товара, иск к платформе возможен |
| Антимонопольное регулирование | При доминировании — эксклюзивные условия могут быть оспорены |
| Валютный контроль при экспансии | Платежи в Узбекистан/Кыргызстан — валютные операции с нерезидентами |
| KYC/AML требования | UGCBoost как платёжный посредник обязан соблюдать |
| Трансграничная передача ПД (ст. 16) | При экспансии — требования к локализации данных |
| Хранение ИИН | Чувствительные ПД — шифрование, минимизация, ответственность за утечку |
| Instagram заблокирован в РФ | Как работать с российскими креаторами при экспансии? |

### Опасные упрощения

- **ОКЭД 82990 как "обходной путь"** для самозанятых — если 500+ креаторов зарегистрируются с одним ОКЭД по рекомендации одной платформы, это привлечёт внимание налоговых органов
- **Пороги модерации** (1K подписчиков, 3% ER) — произвольные числа без казахстанских бенчмарков
- **Солидарная ответственность** за нарушение маркировки — штрафы за каждый факт (100 постов = 100 штрафов)
- **Экспансия в СНГ** — на уровне Wikipedia; не учтены валютный контроль, "иностранные агенты" в РФ, блокировка Instagram в РФ
