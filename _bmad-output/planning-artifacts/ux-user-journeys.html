Флоу 1: Креатор — от заявки до сдачи работы

Креатор Happy path — полный цикл от регистрации до выполнения заказа

Онбординг (лендинг → приложение)

flowchart TD
    A["🌐 Лендинг: увидел UGCBoost"] --> B["📝 Заполняет форму заявки\n(ФИО, ИИН, соцсети, категория, 4 чекбокса ПД)"]
    B --> B1{"✅ Валидация ОК?"}
    B1 -->|"Нет"| B
    B1 -->|"Да"| C["⏳ Экран «Заявка на рассмотрении»"]
    C --> D["🤖 Telegram-бот: нажимает Старт"]
    D --> E["🔐 Верификация аккаунта"]
    E --> E1{"Подтверждено?"}
    E1 -->|"Нет"| E2["❌ Отклонение + объяснение"]
    E1 -->|"Да"| F["📊 Автомодерация LiveDune\n(ER, подписчики, просмотры)"]
    F --> F1{"Метрики ОК?"}
    F1 -->|"Нет"| F2["❌ Автоотклонение + фидбэк"]
    F1 -->|"Да"| G["👁️ Ручная модерация\n(качество контента, визуал, категория)"]
    G --> G1{"Одобрен?"}
    G1 -->|"Нет"| G2["❌ Отклонение + рекомендация"]
    G1 -->|"Да"| H["✍️ TrustMe: подписание договора\n(NDA, обязательства, лицензия)"]
    H --> H1{"Подписал?"}
    H1 -->|"Нет"| H2["🚫 Доступ не открыт"]
    H1 -->|"Да"| I["🎉 Доступ к приложению!"]

Рабочий цикл (кампании → сдача)

flowchart TD
    I["📱 Открыл приложение"] --> J["📋 Лента кампаний\n(Grid 2× / Hero-карточки)"]
    J --> K["👀 Читает бриф кампании\n(фото продукта, ТЗ, дедлайн, бюджет)"]
    K --> L["👆 «Подать заявку»\n(один тап + ознакомление с ТЗ)"]
    L --> M["⏳ Ожидание одобрения бренда"]
    M --> M1{"Одобрен?"}
    M1 -->|"Нет"| M2["📩 Уведомление + другие кампании"]
    M1 -->|"Да"| N["🔔 Push: «Вас одобрили!»"]
    N --> O["⚖️ Экран принятия заказа\n(NDA, условия, лицензия, № договора)"]
    O --> O1{"Принял?"}
    O1 -->|"Нет"| O2["Заказ отклонён"]
    O1 -->|"Да"| P["🔥 Кампания в «Активные»\n+ таймер до дедлайна"]
    P --> Q["⏰ Напоминания\n2нед → 1нед → 3дн → 1день"]
    Q --> R["📎 Сдача работ\n(ссылки + чеклист маркировки)"]
    R --> R1{"Валидация ОК?"}
    R1 -->|"Нет"| R
    R1 -->|"Да"| S["✅ Статус: «Ожидает приёмки»"]

Ключевые точки

  • Подача заявки на кампанию — минимальный барьер (один тап + ознакомление с ТЗ)
  • Принятие заказа — осознанный серьёзный момент (NDA, условия, лицензия)
  • 4 автоматических напоминания по мере приближения дедлайна
  • Момент ценности: push «Вас одобрили!» — заказ пришёл без единого DM

Флоу 2: Бренд — от кампании до результатов

Бренд-менеджер Happy path — создание кампании и управление креаторами

flowchart TD
    A["🏢 Аккаунт создан командой UGCBoost\n(MVP — без саморегистрации)"] --> B["💻 Веб-кабинет: создание кампании"]
    B --> C["📋 Заполнение\n(ТЗ, тип, категория, лимит,\nдедлайн, маркировка, фото)"]
    C --> D["⏳ Кампания на модерации UGCBoost"]
    D --> D1{"Одобрена?"}
    D1 -->|"Нет"| D2["📧 Email: замечания"] --> C
    D1 -->|"Да"| E["📧 Email: «Кампания опубликована»"]
    E --> F["⭐ Ранжированный список заявок\n(top-20/top-50 по метрикам)"]
    F --> G["📊 Карточки креаторов\n(ER, подписчики, просмотры,\nкросс-кампанийный флаг)"]
    G --> H["✅ Выбор top-N\n+ «Одобрить выбранных»"]
    H --> I["📈 Дашборд кампании\n(прогресс-бар выполнения)"]
    I --> J["👀 Просмотр сданных работ"]
    J --> J1{"Соответствует ТЗ?"}
    J1 -->|"Нет"| J2["⚠️ «Не по ТЗ» → эскалация"]
    J1 -->|"Да"| K["✅ Приёмка + акт"]
    K --> L["🖼️ Галерея контента\n(embed-посты + метрики LiveDune)"]

    style A fill:#fef3c7,stroke:#f59e0b
    style F fill:#d1fae5,stroke:#10b981
    style L fill:#d1fae5,stroke:#10b981
    style J2 fill:#fee2e2,stroke:#ef4444

Ключевые точки

  • Вау-момент: ранжированный список с метриками — top-20 за 15 минут вместо 2 недель поиска
  • Pipeline-view: заявки → одобрено → в работе → сдано → принято
  • Массовое одобрение — выбрал чекбоксами, нажал одну кнопку
  • Галерея контента — embed-посты с нативным превью из соцсетей

Флоу 3: Админ — операционный цикл

Админ (Айдана) Утренняя сессия — обработка очередей модерации

flowchart TD
    A["☀️ Утро: открывает админ-панель"] --> B{"📬 Что в очередях?"}
    B --> C["👤 Модерация креаторов"]
    B --> D["📢 Модерация кампаний"]
    B --> E["🚨 Алерты и проблемы"]

    C --> C1["Карточка креатора:\nметрики LiveDune + ссылки на профили\n+ история заявок"]
    C1 --> C2{"Одобрить?"}
    C2 -->|"Да"| C3["✅ Статус обновлён → ожидает TrustMe"]
    C2 -->|"Нет"| C4["❌ Отклонение + фидбэк + причина"]
    C3 --> C5["⏭️ Автопереход к следующему"]
    C4 --> C5

    D --> D1["Бриф кампании:\nТЗ + правила + маркировка"]
    D1 --> D2{"Одобрить?"}
    D2 -->|"Да"| D3["✅ Кампания опубликована"]
    D2 -->|"Нет"| D4["📝 Замечания → бренд"]

    E --> E1["Просрочки / dispute / перегрузка"]
    E1 --> E2["Ручная эскалация\n(Telegram / WhatsApp)"]

    style A fill:#ede9fe,stroke:#8b5cf6
    style C5 fill:#d1fae5,stroke:#10b981
    style D3 fill:#d1fae5,stroke:#10b981
    style E1 fill:#fee2e2,stroke:#ef4444

Ключевые точки

  • Все действия — из одной карточки, без переходов на другие страницы
  • Автопереход к следующему элементу в очереди после решения
  • Кросс-кампанийная карта: кто в скольких кампаниях одновременно
  • 20 заявок за 30 минут вместо 3 часов в Google Sheets

Флоу 4: Отклонение и повторная заявка

Edge case Креатор не прошёл модерацию — путь к исправлению

flowchart TD
    A["📝 Подал заявку, категория: мода"] --> B["📊 Автомодерация LiveDune — ОК"]
    B --> C["👁️ Ручная модерация"]
    C --> D{"Контент соответствует\nкатегории?"}
    D -->|"Нет"| E["❌ Отклонение +\nконкретная рекомендация:\n«Ваш контент — лайфстайл, не мода.\nРекомендуем подать заявку\nв категорию лайфстайл»"]
    E --> F["📩 Уведомление в Telegram:\nпричина + совет"]
    F --> G["🔄 Повторная заявка\nс новой категорией"]
    G --> H["👁️ Модератор видит историю:\nпредыдущее отклонение + причину"]
    H --> I{"Одобрить?"}
    I -->|"Да"| J["✅ → договор → кампании"]
    I -->|"Нет"| K["❌ Повторное отклонение + фидбэк"]
    D -->|"Да"| J

    style E fill:#fef3c7,stroke:#f59e0b
    style J fill:#d1fae5,stroke:#10b981
    style K fill:#fee2e2,stroke:#ef4444

Ключевые точки

  • Не «отказано» — а конкретная рекомендация с путём к исправлению
  • Модератор видит полную историю заявок
  • Тон коммуникации: серьёзно, но справедливо

Флоу 5: Dispute — невыполнение ТЗ

Edge case Креатор сдал работу не по ТЗ

flowchart TD
    A["📎 Работа сдана / дедлайн прошёл"] --> B{"Соответствует ТЗ?"}
    B -->|"Да"| C["✅ Приёмка"]
    B -->|"Нет"| D["⚠️ Бренд/админ: «Не по ТЗ»\n+ комментарий"]
    D --> E["📋 Статус: «Требуется доработка»\n+ новый дедлайн"]
    E --> F["💬 Эскалация: админ → креатор\nчерез Telegram/WhatsApp"]
    F --> G{"Креатор исправил?"}
    G -->|"Да"| H["✅ Повторная приёмка"]
    G -->|"Нет"| I["📌 Инцидент зафиксирован\nв истории креатора"]
    I --> J{"Повторные нарушения?"}
    J -->|"Да"| K["🚫 Бан"]
    J -->|"Нет"| L["⚠️ Предупреждение\n+ снижение рейтинга"]

    style C fill:#d1fae5,stroke:#10b981
    style H fill:#d1fae5,stroke:#10b981
    style K fill:#fee2e2,stroke:#ef4444
    style D fill:#fef3c7,stroke:#f59e0b
    style E fill:#fef3c7,stroke:#f59e0b

Ключевые точки

  • Бренд не разруливает сам — отмечает проблему, платформа координирует
  • Каждый инцидент записывается в историю креатора
  • Прогрессивные последствия: предупреждение → снижение рейтинга → бан

Паттерны поведения

Общие паттерны UX, извлечённые из всех флоу

ПаттернГдеОписание
Одно действие — одна карточкаМодерация, ревью заявокВся информация + кнопки действий на одном экране, без переходов
АвтопереходОчередь модерацииПосле решения — автоматически следующий элемент
Прогрессивная серьёзностьЗаявка → принятие заказаЛёгкий старт (один тап), серьёзный коммит (NDA, условия)
Фидбэк с контекстомОтклонения, ошибкиНе «отказано», а конкретная причина + путь к исправлению
Напоминания-лестницаДедлайны кампаний4 точки: 2нед → 1нед → 3дн → 1день
Pipeline-статусыКампания (бренд)Заявки → одобрено → в работе → сдано → принято
Batch-операцииОдобрение креаторовВыбрать N чекбоксами → одна кнопка «Одобрить выбранных»

Принципы оптимизации флоу

ПринципОписание
Минимум шагов до ценностиКреатор видит кампании сразу после входа. Бренд видит заявки сразу после публикации
Feedback на каждом шагеСтатусы, toast-уведомления, progress indicators — пользователь всегда знает, что происходит
Graceful degradationЕсли шаг не прошёл — конкретное объяснение + путь к исправлению, а не тупик
Batch-операцииБренд одобряет N креаторов за раз. Админ обрабатывает очередь без лишних переходов
