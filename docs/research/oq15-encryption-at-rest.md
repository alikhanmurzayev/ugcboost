# OQ-15: Шифрование персональных данных при хранении (at rest)

## Вопрос

Обязательно ли шифровать персональные данные при хранении (encryption at rest) по Закону РК N 94-V «О персональных данных и их защите»?

## Нормативная база

- **Закон РК N 94-V** «О персональных данных и их защите» от 21.05.2013 (с изменениями по 18.01.2026)
- **Правила осуществления мер по защите ПД** — Приказ МЦРИАП от 12.06.2023 № 179/НҚ (обновлён 06.01.2026)
- **СТ РК 1073-2007** «Средства криптографической защиты информации. Общие технические требования»

## Две категории персональных данных

Закон делит ПД на:

1. **Общедоступные** — данные, на которые не распространяются требования конфиденциальности
2. **Ограниченного доступа** — данные, доступ к которым ограничен законодательством (личная, семейная, налоговая, банковская, медицинская тайна и др.)

## ИИН — двойственный статус

По **официальному разъяснению Комитета по информационной безопасности МЦРИАП**:

- ИИН, ФИО, email **по отдельности** или в простых комбинациях = **общедоступные** данные
- ИИН + ФИО **в связке с** финансовой/медицинской/банковской/налоговой тайной = **ограниченного доступа**

> Примечание: юридический Q&A-сервис dogovor24.kz относит ИИН + ФИО + адрес к «ограниченного доступа» без дополнительных условий. Однако разъяснение МЦРИАП — более авторитетный источник.

## Требования к шифрованию

Правила (Приказ 179/НҚ) устанавливают обязательное шифрование **только для данных ограниченного доступа**:

> «Хранят и передают персональные данные ограниченного доступа с использованием средств криптографической защиты информации, имеющих параметры не ниже **третьего уровня безопасности** согласно стандарту **СТ РК 1073-2007**»

Третий уровень = стойкость к взлому >= 2^120 операций. AES-256 покрывает это требование.

Для **общедоступных** данных — шифрование при хранении **не обязательно**.

## Какие данные хранит UGCBoost MVP

| Данные | Категория |
|--------|-----------|
| ИИН (для договоров через TrustMe) | Общедоступные* |
| ФИО | Общедоступные |
| Телефон, email | Общедоступные |
| Ссылки на соцсети | Общедоступные |

*ИИН без привязки к банковской/медицинской/налоговой тайне = общедоступные по разъяснению МЦРИАП.

MVP — только бартерные кампании, финансовых данных нет.

## Вывод

**Шифрование at rest для UGCBoost MVP юридически НЕ обязательно.**

## Когда потребуется

При добавлении **платных кампаний** (Phase 2+) появятся банковские реквизиты креаторов — данные станут «ограниченного доступа», и шифрование at rest станет обязательным по СТ РК 1073-2007 уровень >= 3.

## Обязательные меры для MVP (вне зависимости от шифрования at rest)

- TLS 1.2+ при передаче данных (in transit)
- Контроль доступа по ролям (RBAC)
- Журналирование событий и действий
- Контроль целостности данных
- Уведомление регулятора об инцидентах в течение 1 рабочего дня (с 01.07.2024)
- Серверы на территории РК
- Для БД > 100 000 записей ПД ограниченного доступа: биометрическая аутентификация (не применимо к MVP)
