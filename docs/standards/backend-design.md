# Архитектура и проектирование [REQUIRED]

## Зависимости через конструктор

Все зависимости инжектируются через `New*()`. Методы `Set*()` для зависимостей запрещены. Структура иммутабельна после создания — никаких проверок на nil при использовании зависимости.

## Конфигурируемые значения — в Config

Всё, что может отличаться между окружениями (local/staging/production) или что может потребовать тюнинга без релиза — в Config через env vars. Hardcoded таймауты, TTL, лимиты запрещены.

Окружение определяется явным `ENVIRONMENT` env var. Поведение (secure cookies, debug logging, test endpoints) зависит от него напрямую, а не вычисляется косвенно из других переменных.

## Разделение типов по пакетам

- `api/` — API-модели (request/response), сгенерированные из OpenAPI
- `domain/` — бизнес-ошибки, бизнес-константы, бизнес-типы. Ручные дубликаты API request/response запрещены — для этого есть кодогенерация
- `repository/` — DB-модели (Row structs с тегами `db:"..."`)

## RepoFactory

`RepoFactory` — экспортируемая stateless структура в пакете `repository`. Без полей, без хранения соединения к БД. Каждый метод принимает `dbutil.DB` и возвращает экспортируемый интерфейс repo.

Каждый сервис объявляет свой интерфейс `RepoFactory` с только нужными ему конструкторами. Сервис хранит `dbutil.TxStarter` (для транзакций) и `RepoFactory` (для создания repos). Детали — в стандарте backend-transactions.
