# Библиотеки вместо велосипедов [REQUIRED]

Инфраструктурный код (конфигурация, lifecycle, логирование, HTTP-утилиты, SQL-утилиты) — использовать проверенные библиотеки. Самописные решения для инфраструктурных задач запрещены.

## Принцип

Бизнес-логика (хендлеры, сервисы, доменные правила) — пишется самостоятельно. Инфраструктура — берётся из библиотек. Каждая строка кастомного инфраструктурного кода — это строка, которую надо поддерживать, тестировать и дебажить. Библиотека с тысячами звёзд уже прошла через это.

## Правило

Перед написанием инфраструктурного кода — проверить, есть ли готовое решение (Awesome Go, pkg.go.dev, npm). Если есть подходящая библиотека с активной поддержкой — использовать её. Если нет — написать своё, но обосновать почему.

Кастомные хелперы и утилиты для стандартных инфраструктурных задач (парсинг env vars, graceful shutdown, и т.п.) — не писать. Ошибки парсинга не игнорировать молча — невалидный конфиг = приложение не стартует.
