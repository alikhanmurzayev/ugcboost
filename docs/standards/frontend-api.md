# Фронтенд: API-слой [REQUIRED]

API-слой — граница между фронтендом и бэкендом. Единообразие, типобезопасность и корректная обработка ошибок.

## API-клиент — сгенерированный из OpenAPI

Все HTTP-запросы — через сгенерированный клиент (openapi-fetch). Пути и типы из спеки, полная типизация, опечатка = ошибка компиляции. Отдельный файл с константами путей не нужен — клиент уже их знает.

Raw `fetch()` запрещён. Исключение: внутри самого API-клиента для инфраструктурных нужд (refresh token) с комментарием почему.

## Query keys — константы с фабрикой

Все query keys определяются в одном файле с фабричными функциями. Строковые литералы query keys в компонентах запрещены. Иерархическая структура ключей обеспечивает правильную инвалидацию кэша (инвалидация родительского ключа затрагивает дочерние).

## Мутации — обработка ошибок обязательна

Каждый `useMutation` обязан иметь `onError` handler. Ошибки мутаций всегда отображаются пользователю — через toast, inline-сообщение или другой UI. Молчаливые ошибки мутаций запрещены.
