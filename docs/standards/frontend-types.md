# Фронтенд: типы и кодогенерация

Единственный источник правды для API-типов — OpenAPI-спецификация. Ручные интерфейсы для API-сущностей запрещены.

## API-типы — только из кодогенерации

- Все типы API-сущностей (User, Brand, Campaign и т.п.) импортируются из `api/generated/schema.ts`
- Ручные `interface`/`type` для API request/response запрещены
- Типы ответов извлекаются из `paths` сгенерированной схемы, ручные обёртки `{ data: { ... } }` запрещены
- Допускается файл `api/types.ts` для реэкспорта с человекочитаемыми именами

## Производные и UI-типы

Типы для форм, локального состояния и UI-логики — допустимы как ручные, но должны быть производными от сгенерированных API-типов через `Pick`, `Omit`, `Partial` и т.п. Полностью ручные типы, дублирующие API-структуру — запрещены.

## Enum-ы из OpenAPI

OpenAPI enum-ы генерируются как string union (`"admin" | "brand_manager"`) — это type-only, в рантайме не существует. Для случаев, когда нужен рантайм-доступ (итерация, маппинг на UI-лейблы), создавать const-объекты рядом с реэкспортом типов.

## Что ревьюить

- [blocker] Ручной `interface` / `type` для API request / response.
- [major] Ручная обёртка `{ data: { ... } }` вместо извлечения из `paths` schema.
- [major] OpenAPI enum используется в рантайме без const-объекта (only type — пропадёт при компиляции).
